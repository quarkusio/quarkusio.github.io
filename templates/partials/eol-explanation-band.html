What Happens When a Release Reaches End-of-Life

{=cdi:releases.policy.eol_description.raw}

No More Vulnerability Fixes

Security vulnerabilities discovered after EOL will not be patched. Your applications remain exposed to known exploits.

Ecosystem Drift

Extensions and libraries drop support for EOL versions, making it progressively harder to update dependencies.

Toolchain Breakage

Build tools, CI systems, and shared libraries may introduce incompatibilities that are never resolved for EOL releases.

Compliance Risks

Industry audits and security frameworks flag unmaintained runtimes, potentially blocking deployments and certifications.

We recommend upgrading to an LTS release. LTS releases are maintained for {=cdi:releases.policy.lts_support_duration.raw} with critical bug fixes and security patches.
